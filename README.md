# GKD-Recruiter: Jointly Modeling Social and Task Heterogeneity for Spatial Crowdsourcing via Graph Knowledge Distillation

This repository contains the official implementation of the paper: **"GKD-Recruiter: Jointly Modeling Social and Task Heterogeneity for Spatial Crowdsourcing via Graph Knowledge Distillation"**.

GKD-Recruiter is a task-aware worker recruitment framework designed to maximize Effective Task Satisfaction (ETS) in spatial crowdsourcing. It models complex worker-task affinity and directional social influence, fusing them via Graph Knowledge Distillation (GKD) and optimizing the seed selection process using Rainbow DQN.

## 📊 Datasets

The experiments are conducted on sampled subgraphs from two real-world Location-Based Social Networks (LBSNs). For clarity in reproducing the results from the paper, please note the following configuration mapping:
* Configurations using **$|V|=3000$** are evaluated on the **Brightkite** dataset.
* Configurations using **$|V|=5000$** are evaluated on the **Gowalla** dataset.

## 🚀 Framework Architecture and Usage

As formulated in the paper, GKD-Recruiter operates in a decoupled **Two-Phase** architecture. This design shifts the heavy graph convolution computations to the offline representation learning phase, enabling highly efficient real-time online inference during the seed selection phase.

### Data Preprocessing
Before starting the training phases, generate the required environment parameters and model features from the source data:
```bash
python data/data_preprocess.py
```

### Phase 1: Graph Representation Learning via GKD
In this phase, the framework extracts structural features from the Social Graph (via IGAT) and the Worker-Task Graph (via RGCN). A Gating Fusion Module acts as the Teacher, and Graph Knowledge Distillation ($L_{KD}$) is applied to align the Student embeddings. This phase effectively filters out spatial-social noise.

To train the representation model and generate distilled embeddings:
```bash
python scripts/train_representation.py
```
**Output:** Upon convergence, the script saves the robust feature representations (`distilled_worker_embeds.pt` and `distilled_task_embeds.pt`) into the `data/pretrain/` directory.

### Phase 2: Seed Selection via Rainbow DQN
In this phase, the RL agent utilizes the fixed distilled embeddings from Phase 1 as the state representation. To navigate the massive combinatorial search space efficiently and avoid saturation traps, the action space is dynamically pruned using the **Top-m Strategy** ($m=5$) prior to the Markov Decision Process.

To train the Rainbow DQN agent for seed selection:
```bash
python scripts/train_gkd.py
```

## 📈 Evaluation
The performance of GKD-Recruiter is evaluated using the **Effective Task Satisfaction (ETS)** metric, which accounts for both social influence propagation and task saturation constraints. Evaluation logic is integrated into the training scripts and can be further explored in `models/evaluate.py`.
